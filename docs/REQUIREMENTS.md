# Redis Desktop Client — 需求文档

## 一、项目概述

### 1.1 项目背景

构建一款**跨平台 Redis 桌面管理工具**，提供类似 Redis Desktop Manager (RESP.app) 的功能体验，支持 macOS、Windows、Linux 三大平台。采用现代化技术栈，以高性能、美观的界面和流畅的操作体验为目标。

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | **Tauri v2** | Rust 后端，负责 Redis 连接管理、数据操作、系统集成 |
| 前端框架 | **Next.js** (App Router) | React 前端，负责 UI 渲染与交互逻辑 |
| UI 组件库 | **HeroUI** | 基于 Tailwind CSS 的组件库，提供美观一致的界面 |
| Redis 客户端 | **redis-rs** | Rust 生态的 Redis 客户端库 |
| 状态管理 | **Zustand** | 轻量级前端状态管理 |
| 数据持久化 | **Tauri Store Plugin** | 本地存储连接配置等数据 |
| 任务管理 | **just** | 跨平台命令运行器，统一项目常用命令入口 |

### 1.3 目标平台

- macOS (Apple Silicon + Intel)
- Windows 10/11
- Linux (Ubuntu/Debian/Fedora)

## 二、功能需求

### 2.1 连接管理

#### 2.1.1 基本连接
- 支持创建、编辑、删除、复制 Redis 连接配置
- 连接参数：主机地址、端口、密码、用户名（ACL）、连接别名
- 支持选择默认数据库 (db0 ~ db15)
- 连接测试功能（Test Connection）
- 连接列表持久化存储（加密保存密码）

#### 2.1.2 高级连接
- **SSH 隧道连接**：支持通过 SSH 跳板机连接 Redis
  - 支持密码认证和密钥认证
  - 支持自定义 SSH 端口
- **SSL/TLS 连接**：支持 TLS 加密连接
  - 支持自签名证书
  - 支持 CA 证书 / 客户端证书配置
- **Redis Sentinel 连接**：支持哨兵模式连接
- **Redis Cluster 连接**：支持集群模式连接

#### 2.1.3 连接组织
- 支持连接分组（文件夹方式组织）
- 支持拖拽排序
- 支持导入/导出连接配置

### 2.2 数据浏览与操作

#### 2.2.1 Key 浏览器
- **树形视图**：按命名空间分隔符（默认 `:`）将 Key 组织成树形结构
- **平铺视图**：列表方式展示所有 Key
- 支持自定义命名空间分隔符
- 分页加载 Key 列表（SCAN 方式，避免阻塞）
- 显示 Key 总数统计
- 支持按模式过滤 Key（通配符搜索）
- 支持多数据库切换 (db0 ~ db15)，显示每个 db 的 Key 数量

#### 2.2.2 数据类型支持

支持 Redis 全部核心数据类型的查看、编辑、新增、删除：

| 数据类型 | 查看 | 编辑 | 新增 | 删除 |
|----------|------|------|------|------|
| **String** | ✅ 支持纯文本 / JSON / 十六进制查看 | ✅ | ✅ | ✅ |
| **Hash** | ✅ 表格展示 field-value | ✅ 逐字段编辑 | ✅ | ✅ 支持删除单个 field |
| **List** | ✅ 带索引的列表展示 | ✅ | ✅ 头部/尾部插入 | ✅ |
| **Set** | ✅ 列表展示 | ✅ | ✅ | ✅ 删除指定成员 |
| **Sorted Set** | ✅ 带分数的表格展示 | ✅ 修改分数 | ✅ | ✅ 删除指定成员 |
| **Stream** | ✅ 消息列表展示 | ❌ | ✅ 添加消息 | ✅ 删除消息/裁剪 |
| **JSON** (RedisJSON) | ✅ 树形/原始 JSON 查看 | ✅ | ✅ | ✅ |

#### 2.2.3 Key 操作
- 查看 Key 的基本信息：类型、TTL、内存占用、编码方式
- 设置/修改/移除 TTL
- 重命名 Key
- 复制 Key（含值）
- 删除 Key（单个/批量，需二次确认）
- 收藏/标记常用 Key

#### 2.2.4 值编辑器
- 多格式查看：纯文本、JSON 格式化、十六进制
- JSON 值自动格式化和语法高亮
- 大文本值支持搜索
- 编辑前后对比（Diff 视图）
- 支持复制值到剪贴板

### 2.3 命令行控制台 (CLI)

- 内置 Redis CLI 终端
- 支持命令自动补全（Redis 命令 + 参数提示）
- 命令历史记录（上/下箭头翻阅）
- 命令执行结果格式化展示
- 支持多 Tab 打开多个 CLI 会话
- 支持对当前连接直接打开 CLI
- 常用命令快捷按钮

### 2.4 服务器信息与监控

#### 2.4.1 服务器信息
- 展示 `INFO` 命令的完整信息，分区块显示：
  - Server：版本、运行时间、OS 等
  - Clients：已连接客户端数、阻塞客户端数等
  - Memory：内存使用量、峰值、碎片率等
  - Stats：命中率、命令处理数等
  - Replication：主从状态
  - Keyspace：各 DB 的 Key 统计

#### 2.4.2 实时监控
- **实时性能图表**：
  - 每秒操作数 (ops/sec)
  - 内存使用趋势
  - 连接客户端数
  - 命中率趋势
- 支持暂停/继续监控
- 支持自定义刷新间隔

#### 2.4.3 慢查询日志
- 展示 `SLOWLOG` 内容
- 显示：执行时间、命令、时间戳
- 支持设置慢查询阈值
- 支持清空慢查询日志

### 2.5 发布/订阅 (Pub/Sub)

- 支持订阅指定频道/模式
- 实时显示收到的消息（时间戳 + 频道 + 内容）
- 支持向指定频道发布消息
- 消息列表支持过滤和搜索
- 支持暂停/继续接收消息

### 2.6 数据导入/导出

- 支持导出选中的 Key 为 JSON 文件
- 支持从 JSON 文件导入 Key
- 支持批量导出（按模式匹配）
- 导入时支持冲突策略选择（跳过/覆盖/重命名）

### 2.7 批量操作

- 批量删除 Key（支持模式匹配）
- 批量设置 TTL
- 批量导出
- 操作前显示影响范围预览
- 操作进度条展示

## 三、非功能需求

### 3.1 性能要求
- 应用启动时间 < 2 秒
- Key 列表加载采用虚拟滚动，支持百万级 Key 浏览
- SCAN 分页加载，单次扫描不阻塞 Redis
- 大值 (> 1MB) 延迟加载，避免界面卡顿
- Rust 后端异步处理所有 Redis 操作

### 3.2 安全要求
- 连接密码本地加密存储
- SSH 密钥文件路径存储，不复制密钥内容
- 支持 TLS 加密连接
- 删除/批量操作需二次确认
- 敏感操作（FLUSHDB/FLUSHALL）需输入确认文字

### 3.3 用户体验
- 深色/浅色主题切换
- 响应式布局（支持窗口缩放）
- 快捷键支持（连接切换、刷新、搜索等）
- 操作结果 Toast 通知
- 加载状态指示器
- 错误信息友好展示

### 3.4 国际化
- **第一期即集成 i18n 框架**，所有 UI 文案从一开始就使用 translation key
- 首期适配中文（zh-CN）和英文（en-US），默认跟随系统语言
- 支持用户手动切换语言，偏好持久化存储
- 架构预留多语言扩展能力，后续可通过添加翻译文件支持更多语言
- 翻译资源按功能模块组织（`common`、`connection`、`browser`、`cli`、`monitor` 等）
- Rust 后端错误信息返回 i18n key，由前端负责翻译展示

### 3.5 CI/CD & 自动更新

#### 3.5.1 GitHub Actions 工作流
- **CI 流水线（每次 Push / PR 触发）**：
  - 前端：ESLint 检查 + TypeScript 类型检查 + 单元测试
  - 后端：`cargo clippy` 代码检查 + `cargo test` 单元测试
  - 跨平台构建验证（macOS / Windows / Linux）
- **Release 流水线（Tag 触发）**：
  - 三平台并行构建 Tauri 应用
  - macOS：`.dmg` 打包 + 代码签名 + 公证（Notarization）
  - Windows：`.msi` / `.exe` 安装包 + 代码签名
  - Linux：`.deb` / `.AppImage` 打包
  - 自动创建 GitHub Release 并上传产物
  - 生成更新清单文件（供自动更新使用）

#### 3.5.2 自动更新
- 集成 **Tauri Updater Plugin**
- 应用启动时检查更新（可配置检查频率）
- 发现新版本时弹窗提示，显示更新日志（Changelog）
- 支持静默下载 + 用户确认安装
- 更新源：GitHub Releases（通过更新清单 JSON）
- 支持用户在设置中关闭自动更新检查

#### 3.5.3 版本管理
- 语义化版本号（Semantic Versioning）：`MAJOR.MINOR.PATCH`
- `Cargo.toml`、`package.json`、`tauri.conf.json` 版本号统一管理
- 基于 Conventional Commits 自动生成 Changelog
- Tag 命名规范：`v1.0.0`
