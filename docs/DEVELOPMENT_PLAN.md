# Redis Desktop Client — 开发计划

> 基于 [需求文档](./REQUIREMENTS.md) 第六章开发阶段规划，将全部功能拆分为 **6 期**迭代开发。  
> 每期目标明确、可独立交付，后一期在前一期基础上扩展。

---

## 总览

| 期数 | 主题 | 核心目标 |
|------|------|----------|
| **Phase 1** | 基础框架搭建 | 初始化项目，集成全部技术栈（含 i18n），搭建布局与主题，建立 CI 基础流水线 |
| **Phase 2** | 连接管理 | 实现连接的完整生命周期管理（CRUD + 测试 + 持久化） |
| **Phase 3** | 数据浏览核心 | Key 浏览器 + 全数据类型增删改查 + TTL 管理 |
| **Phase 4** | CLI 控制台 | 内置命令行终端、自动补全、命令历史 |
| **Phase 5** | 高级功能 | 服务器监控、Pub/Sub、慢查询日志、数据导入导出 |
| **Phase 6** | 高级连接 & 完善 | SSH/TLS/Sentinel/Cluster、国际化、打包发布 |

---

#### 1.5 justfile 命令规划

项目根目录维护一个 `justfile`，作为所有开发命令的统一入口：

```just
# === 开发 ===
dev              # 启动 Tauri 开发模式（前端 + 后端热重载）
dev-web          # 仅启动 Next.js 前端开发服务器
dev-tauri        # 仅启动 Tauri 后端开发

# === 构建 ===
build            # 构建生产版本（Tauri 桌面应用）
build-web        # 仅构建 Next.js 前端
build-debug      # 构建 Debug 版本（含调试符号）

# === 代码检查 ===
lint             # 运行全部代码检查（前端 + 后端）
lint-web         # ESLint + TypeScript 类型检查
lint-rust        # cargo clippy
fmt              # 格式化全部代码（prettier + cargo fmt）
fmt-web          # prettier --write
fmt-rust         # cargo fmt

# === 测试 ===
test             # 运行全部测试（前端 + 后端）
test-web         # 前端单元测试
test-rust        # cargo test

# === 依赖管理 ===
install          # 安装全部依赖（npm install + cargo fetch）
update           # 更新全部依赖
clean            # 清理构建产物

# === 版本 & 发布 ===
version bump     # 同步更新 package.json / Cargo.toml / tauri.conf.json 版本号
changelog        # 基于 Conventional Commits 生成 Changelog
release tag      # 创建 Git Tag 并推送（触发 Release 流水线）

# === 工具 ===
i18n-check       # 检查翻译完整性（key 缺失检测）
icons            # 从源图生成各平台图标资源
```

## Phase 2：连接管理

### 目标

实现连接管理的完整功能闭环。本期结束后，用户可以通过 GUI 创建、编辑、测试连接，并成功连接到 Redis 服务器。

### 任务清单

#### 2.1 连接后端

| # | 任务 | 说明 |
|---|------|------|
| 1 | 实现连接池管理 | 基于 `HashMap<String, ConnectionManager>` 管理多连接 |
| 2 | 实现配置加密模块 | 密码使用 AES 加密存储，密钥绑定本机 |
| 3 | 连接 CRUD 后端命令 | `create_connection`、`update_connection`、`delete_connection`、`list_connections` |
| 4 | 连接/断开命令 | `connect`、`disconnect` 命令，管理连接生命周期 |
| 5 | 连接测试命令 | `test_connection` 命令，PING 验证并返回延迟 |

#### 2.2 连接前端

| # | 任务 | 说明 |
|---|------|------|
| 6 | 新建连接对话框 | 表单：别名、主机、端口、密码、用户名（ACL）、默认 DB |

| 8 | 连接测试 UI | 测试按钮 + 成功/失败/延迟反馈 |
| 9 | 编辑/复制/删除连接 | 右键菜单或操作按钮，编辑复用新建对话框 |

| 12 | 连接状态 Store | Zustand 管理当前活动连接、连接列表状态 |

### 交付物

- ✅ 完整的连接管理功能（增删改查 + 测试 + 分组 + 排序）

## Phase 4：CLI 控制台

### 目标

实现内置 Redis CLI 终端。本期结束后，用户可以直接在应用内执行任意 Redis 命令，支持自动补全和历史记录。

## Phase 5：高级功能

### 目标

实现服务器监控、发布订阅、慢查询日志和数据导入导出。本期结束后，用户可以全面掌握 Redis 服务器运行状态，并方便地进行数据迁移。

#### 5.1 服务器信息

| # | 任务 | 说明 |
|---|------|------|
| 1 | INFO 信息后端 | `get_server_info` 命令，解析 `INFO` 返回值为结构化数据 |
| 2 | 服务器信息页面 | 分 Tab/区块展示 Server、Clients、Memory、Stats、Replication、Keyspace |
| 3 | 关键指标卡片 | 顶部展示：版本号、运行时间、已用内存、客户端数、Key 总数 |
| 4 | 自动刷新 | 支持配置刷新间隔，默认 5 秒 |

#### 5.2 实时监控

| # | 任务 | 说明 |
|---|------|------|

| 7 | 暂停/继续 | 暂停图表更新，保留已有数据 |
| 8 | 刷新间隔配置 | 下拉选择：1s / 2s / 5s / 10s |

#### 5.3 慢查询日志

| # | 任务 | 说明 |

| 9 | 慢查询后端 | `get_slowlog` 命令，返回慢查询列表 |

#### 5.4 发布/订阅 (Pub/Sub)

## 技术风险 & 注意事项

| 风险项 | 影响 | 应对策略 |
|--------|------|----------|
| Next.js SSR 与 Tauri 兼容 | Tauri 仅加载客户端页面 | Next.js 配置 `output: 'export'` 静态导出 |
| 大量 Key 浏览性能 | 百万级 Key 导致内存/渲染压力 | 虚拟滚动 + SCAN 分页 + 树节点延迟加载 |
| SSH 隧道稳定性 | 网络抖动导致隧道断开 | 自动重连机制 + 心跳检测 |
| Cluster 模式路由 | 集群拓扑变更影响连接 | 定期刷新拓扑 + MOVED/ASK 重定向处理 |
| 跨平台兼容性 | 快捷键/文件路径差异 | 使用 Tauri API 抽象平台差异 |
| macOS 签名 & 公证 | 未签名应用无法安装 | 配置 Apple Developer 证书 + CI Secrets 管理 |
| Windows 代码签名 | SmartScreen 拦截未签名安装包 | 购买代码签名证书 + CI 自动签名 |
| 自动更新安全 | 更新包被篡改 | Tauri Updater 内置签名校验机制 |
